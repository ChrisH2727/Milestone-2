× How To Get Started
1. Enter a short Task Description.
2. Select a Task Start Date. This may be the present day, a day in the past or a day in the future.
3. Enter a Task Best Case Duration, Task Most Likely Case Duration and Task Wost Case Duration into the Task Table below.
4. Enter as many tasks as you need to. All tasks have an automatic finish to start relationship.
5. Select the Non Project Work Days allocted to your team. Note that Saturday & Sunday are set by default as Non Working Days.
6.Select the number of simulation runs required. The larger the number of simulation runs selected, the longer the simulation will take to run, however the outcome will be more accurate.
7. When you are ready click the Start button.
8. You can add more tasks, change task durations and change Non Project Work Days at any time and the task table will automatically update, but you must click the Start button to run the simulation.
9. If you select every day of the week as a non project work day, then the programme will automatically allocate Monday as the only work day.
10. Click the Restart button to clear the Task Entry Table.

How To Read The Charts
On clicking the Start button two charts are generated.
The Project Timeline chart show the task best case, most likely case and worst case dates graphically. The worst case finish date of a task determines the start date of the next task. At a project level this chart shows the Timeline for the project if everything that could go wrong does go wrong. In practical terms this is unlikely to be the case. Some project risks will mature into problems and others will become obsolete over the duration of the project.
The Project Risk profile chart takes this a stage further and provides a statistical analysis of the task best case, most likely case and worst case durations and then provides probability of completing a project by a given date. The final date provided by the risk profile is always earlier or the same as the final date given by the Project Timeline chart.
In a project not everything that can go wrong will go wrong..
